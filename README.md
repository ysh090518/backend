# GET, POST, PUT, DELETE, PATCH

### GET

데이터를 읽거나 검색할 때 사용되는 메소드
서버에 전달하고 싶은 데이터는 query(쿼리 파라미터, 쿼리 스트링)를 통해서 전달
서버의 데이터 변경이 일어나지 않음

### POST

새로운 리소스를 생성하거나 기존 리소스를 수정하기 위해 서버에 데이터를 보낼 때 사용
서버의 데이터 변경이 일어날 수 있음

### PUT
리소스 생성, 업데이트를 위해 사용
클라이언트가 리소스 위치를 식별해서 URL 지정

### DELETE
지정된 리소스를 삭제

### PATCH
리소스의 일부분을 업데이트
지원 안하는 서버에서는 POST 메서드 사용

# 콜백함수

함수 안에서 함수를 호출시키는 함수

# 배열매소드
### forEach()
배열의 각 원소별로 지정된 함수를 실행

### map()
배열의 각 원소별로 지정된 함수를 실행한 결과로 구성된 새로운 배열을 반환

### filter()
지정된 함수의 결과 값을 true로 만드는 원소들로만 구성된 별도의 배열을 반환

### find
배열의 모든 요소들을 돌면서 전달한 콜백함수의 인자로 넣고 호출, 콜백 함수의 반환된 값이 true이면 그 즉시 멈추고 해당 요소를 리턴

### findIndex()
배열에서 특정 조건을 만족하는 첫 번째 요소의 인덱스를 반환

### includes()
특정 값을 포함하는 지 여부 확인

### indexOf()
배열에서 특정 요소의 인덱스를 찾는 데 사용

### push()
배열 뒷부분에 값을 삽입

### pop()
배열 뒷부분의 값을 삭제

### unshift()
배열 앞부분에 값을 삽입

### shift()
배열 앞부분의 값을 삭제

### sort()
배열의 원소를 알파벳순으로 순서로 정렬

### reverse()
배열의 아이템 순서를 뒤집음

### join()
배열 원소 전부를 하나의 문자열로 합침.

### slice()
배열의 특정한 부분을 리턴

### splice()
배열의 요소를 추가, 삭제 또는 교체
